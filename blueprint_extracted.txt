1. Product Vision
Mission Statement
WorkShift empowers daily workers to find dignified, well-paying work in minutes and enables businesses to solve their flexible labor needs instantly — creating economic opportunity at scale.
Core Problem Solved
The global daily labor market is broken:
Workers wait at street corners hoping to be picked — unreliable, undignified, and unsafe.
Employers spend hours calling contacts to fill urgent labor gaps, losing productivity and money.
Payments are cash-based, creating theft risk, dispute potential, and zero financial record for workers.
No trust or verification layer exists — employers can't vet workers, workers can't vet employers.
Value Proposition
Stakeholder
Value Delivered
For Workers
Find verified, well-paying day jobs in minutes via mobile app. Secure digital payment. Build a portable reputation.
For Employers
Hire verified, rated workers on-demand. Post a job, accept applicants, pay digitally — all in one platform.
For the Economy
Formalise the informal labor sector. Enable financial inclusion for millions of unbanked workers.

Real-Life Scenarios
Scenario 1: Emergency Plumbing
A restaurant owner discovers a burst pipe at 6am before breakfast service. He opens WorkShift, posts an emergency plumbing job, sets a premium rate, and within 8 minutes accepts a verified master plumber 2km away. The job is done by 8am. Payment is auto-released from escrow.
Scenario 2: Harvest Season
A mango farm needs 50 harvest workers for 3 days. The farm manager posts a bulk job with location, daily wage, and requirements. WorkShift's AI suggests 72 qualified workers nearby. The manager accepts 50, who arrive the next morning. Attendance is tracked via QR code.
Scenario 3: Hotel Overflow
A 5-star hotel is overbooked for a conference weekend and needs 20 extra cleaning staff for 2 days. WorkShift's employer dashboard lets them filter by hotel-service skill, hygiene rating, and proximity. Workers are booked and briefed via in-app chat within 30 minutes.

6. Database Design
Schema Overview
PostgreSQL is the primary relational database. All tables use UUID primary keys for scalability and security. Soft deletes are implemented via deleted_at timestamps.
Core SQL Schema
-- USERS (base authentication table)
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email         VARCHAR(255) UNIQUE,
  phone         VARCHAR(20)  UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  role          VARCHAR(20)  NOT NULL CHECK (role IN ('worker','employer','admin','super_admin')),
  is_verified   BOOLEAN DEFAULT FALSE,
  is_active     BOOLEAN DEFAULT TRUE,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW(),
  deleted_at    TIMESTAMPTZ
);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_phone ON users(phone);
CREATE INDEX idx_users_role  ON users(role);

-- WORKERS
CREATE TABLE workers (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID UNIQUE REFERENCES users(id) ON DELETE CASCADE,
  first_name       VARCHAR(100) NOT NULL,
  last_name        VARCHAR(100) NOT NULL,
  bio              TEXT,
  avatar_url       TEXT,
  id_document_url  TEXT,
  id_verified      BOOLEAN DEFAULT FALSE,
  availability     VARCHAR(20) DEFAULT 'offline' CHECK (availability IN ('available','busy','offline')),
  latitude         DECIMAL(10,8),
  longitude        DECIMAL(11,8),
  location_point   GEOGRAPHY(POINT,4326),
  avg_rating       DECIMAL(3,2) DEFAULT 0,
  total_jobs       INTEGER DEFAULT 0,
  total_earnings   DECIMAL(14,2) DEFAULT 0,
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  updated_at       TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_workers_user_id ON workers(user_id);
CREATE INDEX idx_workers_location ON workers USING GIST(location_point);
CREATE INDEX idx_workers_availability ON workers(availability);

-- EMPLOYERS
CREATE TABLE employers (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID UNIQUE REFERENCES users(id) ON DELETE CASCADE,
  company_name     VARCHAR(255) NOT NULL,
  company_logo_url TEXT,
  industry         VARCHAR(100),
  description      TEXT,
  reg_doc_url      TEXT,
  is_verified      BOOLEAN DEFAULT FALSE,
  avg_rating       DECIMAL(3,2) DEFAULT 0,
  total_jobs_posted INTEGER DEFAULT 0,
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  updated_at       TIMESTAMPTZ DEFAULT NOW()
);

-- SKILLS
CREATE TABLE skills (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name       VARCHAR(100) UNIQUE NOT NULL,
  category   VARCHAR(100),
  icon_url   TEXT
);

-- WORKER_SKILLS
CREATE TABLE worker_skills (
  worker_id       UUID REFERENCES workers(id) ON DELETE CASCADE,
  skill_id        UUID REFERENCES skills(id) ON DELETE CASCADE,
  experience_level VARCHAR(20) CHECK (experience_level IN ('beginner','intermediate','expert')),
  PRIMARY KEY (worker_id, skill_id)
);

-- JOB_CATEGORIES
CREATE TABLE job_categories (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        VARCHAR(100) UNIQUE NOT NULL,
  description TEXT,
  icon_url    TEXT,
  commission_pct DECIMAL(5,2) DEFAULT 10.00
);

-- JOBS
CREATE TABLE jobs (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  employer_id     UUID REFERENCES employers(id) ON DELETE SET NULL,
  category_id     UUID REFERENCES job_categories(id),
  title           VARCHAR(255) NOT NULL,
  description     TEXT NOT NULL,
  status          VARCHAR(30) DEFAULT 'draft'
                  CHECK (status IN ('draft','open','ongoing','completed','cancelled','disputed')),
  pay_type        VARCHAR(20) CHECK (pay_type IN ('hourly','daily','fixed')),
  pay_rate        DECIMAL(10,2) NOT NULL,
  currency        VARCHAR(10) DEFAULT 'USD',
  workers_needed  INTEGER DEFAULT 1,
  workers_hired   INTEGER DEFAULT 0,
  start_time      TIMESTAMPTZ NOT NULL,
  end_time        TIMESTAMPTZ,
  latitude        DECIMAL(10,8),
  longitude       DECIMAL(11,8),
  location_point  GEOGRAPHY(POINT,4326),
  address         TEXT,
  is_urgent       BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_jobs_employer ON jobs(employer_id);
CREATE INDEX idx_jobs_status   ON jobs(status);
CREATE INDEX idx_jobs_location ON jobs USING GIST(location_point);
CREATE INDEX idx_jobs_category ON jobs(category_id);

-- APPLICATIONS
CREATE TABLE applications (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id       UUID REFERENCES jobs(id) ON DELETE CASCADE,
  worker_id    UUID REFERENCES workers(id) ON DELETE CASCADE,
  status       VARCHAR(30) DEFAULT 'pending'
               CHECK (status IN ('pending','accepted','rejected','withdrawn','completed','cancelled')),
  cover_message TEXT,
  checkin_time TIMESTAMPTZ,
  checkout_time TIMESTAMPTZ,
  proof_url    TEXT,
  created_at   TIMESTAMPTZ DEFAULT NOW(),
  updated_at   TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(job_id, worker_id)
);

-- WALLETS
CREATE TABLE wallets (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID UNIQUE REFERENCES users(id) ON DELETE CASCADE,
  balance     DECIMAL(14,2) DEFAULT 0 CHECK (balance >= 0),
  currency    VARCHAR(10) DEFAULT 'USD',
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);

-- PAYMENTS
CREATE TABLE payments (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id          UUID REFERENCES jobs(id),
  employer_id     UUID REFERENCES employers(id),
  worker_id       UUID REFERENCES workers(id),
  gross_amount    DECIMAL(14,2) NOT NULL,
  commission_pct  DECIMAL(5,2),
  commission_amt  DECIMAL(14,2),
  net_amount      DECIMAL(14,2) NOT NULL,
  status          VARCHAR(30) DEFAULT 'pending'
                  CHECK (status IN ('pending','escrowed','released','refunded','disputed')),
  gateway         VARCHAR(50),
  gateway_txn_id  VARCHAR(255),
  paid_at         TIMESTAMPTZ,
  released_at     TIMESTAMPTZ,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- WALLET_TRANSACTIONS
CREATE TABLE wallet_transactions (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  wallet_id   UUID REFERENCES wallets(id) ON DELETE CASCADE,
  type        VARCHAR(30) CHECK (type IN ('credit','debit','withdrawal','refund','escrow')),
  amount      DECIMAL(14,2) NOT NULL,
  reference   TEXT,
  description TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- REVIEWS
CREATE TABLE reviews (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id       UUID REFERENCES jobs(id),
  reviewer_id  UUID REFERENCES users(id),
  reviewee_id  UUID REFERENCES users(id),
  rating       SMALLINT NOT NULL CHECK (rating BETWEEN 1 AND 5),
  comment      TEXT,
  created_at   TIMESTAMPTZ DEFAULT NOW()
);

-- NOTIFICATIONS
CREATE TABLE notifications (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id) ON DELETE CASCADE,
  type        VARCHAR(50),
  title       VARCHAR(255),
  body        TEXT,
  data        JSONB,
  is_read     BOOLEAN DEFAULT FALSE,

-- MESSAGES
CREATE TABLE messages (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  conversation_id UUID NOT NULL,
  sender_id    UUID REFERENCES users(id),
  content      TEXT NOT NULL,
  media_url    TEXT,
  is_read      BOOLEAN DEFAULT FALSE,

CREATE TABLE disputes (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),

CREATE TABLE audit_logs (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id),
  action      VARCHAR(100) NOT NULL,
  resource    VARCHAR(100),
  resource_id UUID,

  metadata    JSONB,

CREATE INDEX idx_audit_logs_user ON audit_logs(user_id, created_at);

Authentication Controller
// controllers/auth.controller.js
const bcrypt = require('bcrypt');
const jwt = require('jsonwebtoken');
const { prisma } = require('../config/database');
const { sendOTP, verifyOTP } = require('../services/otp.service');

const register = async (req, res, next) => {
  try {
    const { email, phone, password, role } = req.body;
    const existing = await prisma.user.findFirst({
      where: { OR: [{ email }, { phone }] }
    });
    if (existing) return res.status(409).json({ error: 'User already exists' });

    const password_hash = await bcrypt.hash(password, 12);
    const user = await prisma.user.create({
      data: { email, phone, password_hash, role }
    });
    await sendOTP(phone);  // Send OTP via Twilio

    res.status(201).json({ message: 'OTP sent. Please verify.', userId: user.id });
  } catch (err) { next(err); }
};

const login = async (req, res, next) => {
  try {
    const { email, password } = req.body;
    const user = await prisma.user.findUnique({ where: { email } });
    if (!user || !await bcrypt.compare(password, user.password_hash))
      return res.status(401).json({ error: 'Invalid credentials' });

    const accessToken = jwt.sign(
      { userId: user.id, role: user.role },
      process.env.JWT_SECRET, { expiresIn: '15m' }
    );
    const refreshToken = jwt.sign(
      { userId: user.id }, process.env.JWT_REFRESH_SECRET, { expiresIn: '7d' }
    );
    // Store refresh token in Redis with TTL
    await redis.setex(`refresh:${user.id}`, 604800, refreshToken);

    res.json({ accessToken, refreshToken, user: { id: user.id, role: user.role } });
  } catch (err) { next(err); }
};

const verifyPhone = async (req, res, next) => {
  try {
    const { phone, otp } = req.body;
    const valid = await verifyOTP(phone, otp);
    if (!valid) return res.status(400).json({ error: 'Invalid or expired OTP' });

    await prisma.user.update({ where: { phone }, data: { is_verified: true } });
    res.json({ message: 'Phone verified successfully' });
  } catch (err) { next(err); }
};

module.exports = { register, login, verifyPhone };

// controllers/job.controller.js
const { prisma } = require('../config/database');
const { geocode } = require('../utils/geocode');
const { notifyNearbyWorkers } = require('../services/matching.service');

const createJob = async (req, res, next) => {
  try {
    const { title, description, category_id, pay_type, pay_rate,
            workers_needed, start_time, end_time, address, is_urgent } = req.body;
    const employer = await prisma.employer.findUnique({
      where: { user_id: req.user.userId }
    });

    const { lat, lng } = await geocode(address);
    const job = await prisma.job.create({
      data: {
        employer_id: employer.id, category_id, title, description,
        pay_type, pay_rate, workers_needed, start_time, end_time,
        address, latitude: lat, longitude: lng,
        location_point: `SRID=4326;POINT(${lng} ${lat})`,
        is_urgent, status: 'open'
      }
    });

    // Notify nearby available workers asynchronously via queue
    await matchingQueue.add('notify-workers', {
      jobId: job.id, lat, lng, categoryId: category_id, radiusKm: 10
    });

    res.status(201).json({ job });
  } catch (err) { next(err); }
};

const getNearbyJobs = async (req, res, next) => {
  try {
    const { lat, lng, radius = 10, category_id, page = 1, limit = 20 } = req.query;
    const offset = (page - 1) * limit;

    const jobs = await prisma.$queryRaw`
      SELECT j.*, ST_Distance(j.location_point, ST_GeogFromText(
        'SRID=4326;POINT(' || ${lng} || ' ' || ${lat} || ')')) / 1000 AS distance_km
      FROM jobs j
      WHERE j.status = 'open'
        AND ST_DWithin(j.location_point,
          ST_GeogFromText('SRID=4326;POINT(' || ${lng} || ' ' || ${lat} || ')'),
          ${radius} * 1000)
        ${category_id ? Prisma.sql`AND j.category_id = ${category_id}` : Prisma.empty}
      ORDER BY distance_km ASC
      LIMIT ${Number(limit)} OFFSET ${offset}
    `;
    res.json({ jobs });
  } catch (err) { next(err); }
};

module.exports = { createJob, getNearbyJobs };
Wallet Service
// services/wallet.service.js
const { prisma } = require('../config/database');

const creditWallet = async (userId, amount, description, reference) => {
  return await prisma.$transaction(async (tx) => {
    const wallet = await tx.wallet.update({
      where: { user_id: userId },
      data: { balance: { increment: amount } }
    });
    await tx.walletTransaction.create({
      data: { wallet_id: wallet.id, type: 'credit', amount, description, reference }
    });
    return wallet;
  });
};

const debitWallet = async (userId, amount, description, reference) => {
  return await prisma.$transaction(async (tx) => {
    const wallet = await tx.wallet.findUnique({ where: { user_id: userId } });
    if (wallet.balance < amount) throw new Error('Insufficient balance');

    const updated = await tx.wallet.update({
      where: { user_id: userId },
      data: { balance: { decrement: amount } }
    });
    await tx.walletTransaction.create({
      data: { wallet_id: wallet.id, type: 'debit', amount, description, reference }
    });
    return updated;
  });
};

const releaseEscrow = async (paymentId) => {
  const payment = await prisma.payment.findUnique({
    where: { id: paymentId }, include: { job: true }
  });
  if (payment.status !== 'escrowed') throw new Error('Payment not in escrow');

  // Credit worker net amount, update payment status
  await creditWallet(payment.worker_id, payment.net_amount,
    `Payment for job: ${payment.job.title}`, paymentId);

  await prisma.payment.update({
    where: { id: paymentId },
    data: { status: 'released', released_at: new Date() }
  });
};

module.exports = { creditWallet, debitWallet, releaseEscrow };

// socket/index.js — Socket.io server setup
const { Server } = require('socket.io');
const { verifySocketJWT } = require('../middlewares/auth.middleware');

module.exports = (httpServer) => {
  const io = new Server(httpServer, {
    cors: { origin: process.env.FRONTEND_URL, methods: ['GET','POST'] },
    transports: ['websocket', 'polling']
  });

  io.use(verifySocketJWT);  // Verify JWT on handshake

  io.on('connection', (socket) => {
    const userId = socket.data.userId;
    socket.join(`user:${userId}`);  // Personal room

    // Join job room when viewing a job
    socket.on('join:job', (jobId) => socket.join(`job:${jobId}`));
    socket.on('leave:job', (jobId) => socket.leave(`job:${jobId}`));

    // Chat messaging
    socket.on('chat:send', async ({ conversationId, content }) => {
      const msg = await saveMessage({ conversationId, senderId: userId, content });
      io.to(`conversation:${conversationId}`).emit('chat:message', msg);
    });

    socket.on('disconnect', () => {
      console.log(`User ${userId} disconnected`);
    });
  });

  return io;
};

Example: Axios Service with Auth Interceptor
// services/api.js
import axios from 'axios';
import { useAuthStore } from '@/store/authStore';

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  headers: { 'Content-Type': 'application/json' }
});

api.interceptors.request.use(config => {
  const token = useAuthStore.getState().accessToken;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

api.interceptors.response.use(
  res => res,
  async err => {
    if (err.response?.status === 401) {
      // Try refresh
      const refreshToken = useAuthStore.getState().refreshToken;
      const { data } = await axios.post('/auth/refresh', { refreshToken });
      useAuthStore.getState().setTokens(data.accessToken, data.refreshToken);
      return api(err.config);
    }
    return Promise.reject(err);
  }
);

export default api;

12. Security Architecture
Authentication & Authorization
Control
Implementation
JWT Tokens
Short-lived access tokens (15min) + long-lived refresh tokens (7 days) stored in Redis. Refresh token rotation on each use.
Password Hashing
bcrypt with cost factor 12. Passwords never logged or transmitted in plaintext.
OTP Verification
6-digit time-based OTP via Twilio SMS. Expires in 10 minutes. Max 5 attempts before lockout.
Role-Based Access
Middleware enforces role requirements on every protected route. Roles: worker, employer, admin, super_admin.
Social Auth
OAuth2 PKCE flow for Google Sign-In. No password stored for social accounts.
API Security
Rate limiting: 100 req/15min per IP (unauthenticated), 500 req/15min per user (authenticated)
Helmet.js for security headers (CSP, HSTS, X-Frame-Options, etc.)
CORS whitelist: only approved frontend origins
Input validation with Zod on all request bodies — prevents injection attacks
SQL injection prevention: Prisma parameterized queries exclusively
File upload validation: type whitelist (jpg, png, pdf), max 5MB, virus scan via ClamAV
Payment Security
All payment processing via Stripe — WorkShift never stores raw card data (PCI DSS compliant)
Escrow model: funds held in platform Stripe account, released only on confirmed completion
Webhook signature verification for all Stripe events
Payout whitelisting: bank accounts require admin approval for first payout
Data Security
All data encrypted at rest (AES-256) in RDS and S3
All traffic encrypted in transit (TLS 1.3)
PII fields (phone, email, documents) stored with field-level encryption via AWS KMS
GDPR/data privacy: users can request data export or deletion (right to erasure)
Audit logs retained for 2 years — immutable, append-only table

14. AI Features
AI Worker-Job Matching
The matching engine scores every available worker against a new job posting and notifies the top N candidates. Scoring uses a weighted model:
Signal
Description
Skill Match
Jaccard similarity between job required skills and worker skill set. Weight: 35%
Distance
Haversine distance. Workers within 2km score 100%, decaying to 0% at 20km. Weight: 25%
Rating Score
Worker avg_rating normalized to 0-1. Weight: 20%
Reliability
Job completion rate × response rate. Weight: 10%
Availability
Binary: available=1, otherwise excluded. Weight: filter
Job History
Workers who've completed similar category jobs get 10% boost. Weight: 10%
Matching Formula
match_score = (skill_sim × 0.35) + (distance_score × 0.25) + (rating_norm × 0.20) + (reliability × 0.10) + (history_bonus × 0.10). Workers with match_score >= 0.65 are notified.
Worker Reputation Scoring
A dynamic reputation score (0-100) is maintained per worker and updated after each job. It influences search ranking and matching priority.
Component
Calculation
Rating Average
Weighted average of last 50 reviews (recent reviews weighted 2x). Max: 40 points
Completion Rate
Accepted jobs completed / total accepted jobs. Max: 25 points
On-time Rate
Check-in within 15min of start time. Max: 15 points
Profile Completeness
Photo, bio, skills, verified ID → up to 10 points
Response Rate
Application/invitation responded within 2hrs. Max: 10 points
Fraud Detection
An ML-based anomaly detection system monitors platform activity in real-time:
Velocity checks: multiple accounts from same device ID or IP address
Payment anomalies: rapid withdrawal requests after account creation
Review fraud: employer and worker with no prior interaction submitting reviews
Location spoofing: check-in GPS coordinates don't match job site
Identity duplication: OCR on uploaded IDs checks for duplicate document numbers
Flagged accounts are auto-suspended pending admin review. Admins receive real-time alerts via dashboard.
Job Demand Prediction
A time-series forecasting model (Prophet / LSTM) predicts demand by job category, day, and location. Used for:
Surge pricing suggestions to employers when demand exceeds supply
Proactive worker notifications: 'High demand for plumbers tomorrow in your area'
Inventory planning: alert admins to worker supply gaps in specific regions
